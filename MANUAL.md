## 1. Πώς μπαίνω

### Διεύθυνση

Άνοιξε browser (Chrome / Safari / Firefox) και πήγαινε στη διεύθυνση:

👉 **https://www.kokalamarias.gr/admin/**

### Σύνδεση

1. Θα δεις το λογότυπο του ΚΟΚ και ένα κουμπί **«Login with GitHub»**
2. Πάτα το κουμπί
3. Ανοίγει ένα παράθυρο GitHub. Συνδέσου με τον λογαριασμό του ομίλου: **`KOKalamarias`**
4. Την πρώτη φορά θα σου ζητήσει **«Authorize»** — πάτα το
5. Το παράθυρο κλείνει αυτόματα και μπαίνεις στο CMS

⚠️ **Σημαντικό**: Μόνο όσοι έχουν πρόσβαση στον λογαριασμό **KOKalamarias** στο GitHub μπορούν να μπουν. Αν θες να δώσεις πρόσβαση σε άλλο άτομο, μοιράσου τον κωδικό του GitHub λογαριασμού (ή μίλα στον υπεύθυνο τεχνικής υποστήριξης).

## 2. Η εικόνα του CMS

Όταν μπαίνεις βλέπεις:

- **Πάνω-αριστερά**: «Contents» (περιεχόμενο) και «Media» (φωτογραφίες)
- **Αριστερή στήλη — Collections**: η λίστα ενοτήτων:
  - 📰 **Νέα & Ανακοινώσεις** — άρθρα του site
  - 🏊 **Αθλητές / Πρωταθλητές** — οι αθλητές με διακρίσεις
  - 🏛 **Διοικητικό Συμβούλιο** — μέλη Δ.Σ.
  - 🏊‍♂️ **Τμήματα** — επίπεδα κολύμβησης
  - 📝 **Κείμενα Σελίδων** — κεντρικά κείμενα (Hero, About, Επικοινωνία κλπ)
- **Κέντρο**: αυτό που έχεις επιλέξει
- **Πάνω-δεξιά**: το όνομα του ομίλου και το avatar — από εκεί κάνεις **logout**

## 3. Νέο Άρθρο

### Βήμα-βήμα

**1)** Πάτα **«📰 Νέα & Ανακοινώσεις»** στα αριστερά.

**2)** Πάτα **«Λίστα Άρθρων»** στο κέντρο.

**3)** Θα ανοίξει ο editor. Στην ενότητα **«Άρθρα (τα νεότερα πρώτα)»** βλέπεις όλα τα υπάρχοντα άρθρα.

**4)** Κάτω από τη λίστα ή σε κάποια άκρη, υπάρχει κουμπί **«Add Άρθρα»** (ή «+ Προσθήκη»). Πάτα το.

**5)** Εμφανίζεται **νέα κενή καταχώρηση**. Συμπλήρωσε τα πεδία:

| Πεδίο | Τι βάζω |
|---|---|
| **Τίτλος** | Π.χ. *Νίκη στο Πανελλήνιο 2026* |
| **Τίτλος (Αγγλικά)** | Π.χ. *Victory at the 2026 Greek Championship* (προαιρετικό αλλά συνιστάται) |
| **URL slug** | Π.χ. `niki-panellinio-2026` — δες ενότητα [Slugs](#-τι-είναι-το-url-slug) παρακάτω |
| **Ημερομηνία** | Επίλεξε από ημερολόγιο. Είναι η ημερομηνία του γεγονότος |
| **Ετικέτα ημερομηνίας** | Π.χ. *Καλαμαριά · Μάιος 2026* (όπως θες να εμφανίζεται ως μικρή ετικέτα) |
| **Ετικέτα ημερομηνίας (Αγγλικά)** | Π.χ. *Kalamaria · May 2026* |
| **Κατηγορία** | Επίλεξε από: Πρωτάθλημα, Παγκόσμιο, Ευρωπαϊκό, Ρεκόρ, Εθνικές Ομάδες, Εγγραφές, Εκδήλωση, Κοινωνικό |
| **Κύρια φωτογραφία** | Προαιρετικό. Πάτα **«Choose Image»** για να ανεβάσεις/επιλέξεις φωτό |
| **Σύντομη περίληψη** | 1-2 προτάσεις που θα φαίνονται στη λίστα νέων |
| **Σύντομη περίληψη (Αγγλικά)** | Η αγγλική μετάφραση |
| **Πλήρες κείμενο** | Το βασικό κείμενο του άρθρου (markdown) |
| **Πλήρες κείμενο (Αγγλικά)** | Η αγγλική έκδοση |
| **Featured** | ⭐ Αν θες αυτό να φανεί ως μεγάλη κάρτα με φωτογραφία στην κορυφή. **Μόνο 1-2 άρθρα** ταυτόχρονα featured! |
| **Δημοσιευμένο** | ✅ Άστο τσεκαρισμένο για να φαίνεται. Αν είναι ξετσέκαρο, κρύβεται από το site |

**6)** Σύρε το νέο άρθρο **στην κορυφή της λίστας** (αν θέλεις να εμφανίζεται πρώτο). Για να σύρεις, πάτα και κράτα τη δεξιά πλευρά της εγγραφής όπου υπάρχει εικονίδιο ⋮⋮ (κουκίδες).

**7)** Πάτα **«Publish»** πάνω-δεξιά → **«Publish now»**. (Δες ενότητα [10](#10-publish) για λεπτομέρειες.)

**8)** Σε ~1-2 λεπτά οι αλλαγές θα εμφανιστούν στο **https://www.kokalamarias.gr/news**.

## 4. Επεξεργασία Άρθρου

**1)** Πάτα **«📰 Νέα & Ανακοινώσεις»** → **«Λίστα Άρθρων»**.

**2)** Βρες το άρθρο που θες να αλλάξεις. Πάτα πάνω στον τίτλο για να ξεδιπλωθεί.

**3)** Άλλαξε όποια πεδία θες.

**4)** **Publish** → **Publish now**.

### Διαγραφή άρθρου

Στο πλάι κάθε άρθρου υπάρχει εικονίδιο **κάδου 🗑** ή «X». Πάτα το και επιβεβαίωσε.

⚠️ Η διαγραφή είναι μόνιμη μετά το Publish! Εναλλακτικά μπορείς να ξετσεκάρεις το **«Δημοσιευμένο»** ώστε να κρυφτεί χωρίς να χαθεί.

## 5. Φωτογραφίες

### Πώς ανεβάζω καινούργια φωτό

**Μέθοδος Α — Όταν φτιάχνεις άρθρο/εγγραφή:**
1. Στο πεδίο **«Κύρια φωτογραφία»** πάτα **«Choose Image»**
2. Στο παράθυρο που ανοίγει, πάτα **«Upload»** πάνω-δεξιά
3. Επίλεξε εικόνα από τον υπολογιστή σου
4. Μόλις ανέβει, **πάτα πάνω της** και πάτα **«Choose Selected»**

**Μέθοδος Β — Γενικά (όλες οι φωτογραφίες μαζί):**
1. Πάτα **«Media»** πάνω-αριστερά
2. Πάτα **«Upload»** πάνω-δεξιά
3. Επίλεξε μία ή πολλές εικόνες
4. Έτοιμο — οι φωτό σώζονται στο φάκελο `images/uploads/`

### Συμβουλές για φωτογραφίες

- ✅ Μέγεθος: ιδανικά **1600×900 px** ή κοντά (για κύριες φωτό άρθρων)
- ✅ Format: **.jpg** για φωτογραφίες, **.png** για graphics/λογότυπα
- ✅ Συμπίεση: όχι πάνω από **500KB ανά φωτό** (χρησιμοποίησε https://tinypng.com αν είναι μεγαλύτερη)
- ❌ Όχι φωτογραφίες με κάθετο orientation για κύριες (cropped από το site)

## 10. Publish

### Πώς γίνεται δημοσίευση

1. Αφού ολοκληρώσεις τις αλλαγές, πάτα **«Publish»** πάνω-δεξιά
2. Στο μενού επίλεξε **«Publish now»**
3. Θα φύγεις από τη φόρμα και θα γυρίσεις στη λίστα
4. **Σε ~1-2 λεπτά** οι αλλαγές εμφανίζονται στο πραγματικό site

### Γιατί καθυστερεί 1-2 λεπτά;

Επειδή κάθε αλλαγή τσεκαρισμένη με Publish:
- Αποθηκεύεται στο GitHub (το αποθετήριο του site)
- Το Cloudflare κάνει αυτόματο rebuild του site
- Όταν τελειώσει το build → οι αλλαγές γίνονται live

### Save vs Publish

- **Save** (αν εμφανιστεί): απλώς σώζει τις αλλαγές προσωρινά, **δεν** τις δημοσιεύει
- **Publish**: σώζει ΚΑΙ δημοσιεύει

## 🔗 Τι είναι το URL slug

Το **URL slug** είναι ένα μικρό κομμάτι κειμένου που μπαίνει στο τέλος του URL για να δίνει σε κάθε άρθρο/αθλητή/τμήμα τη **δική του διεύθυνση**.

**Παράδειγμα:**
- Slug: `5o-protathlima`
- Τελική διεύθυνση: `https://www.kokalamarias.gr/news/5o-protathlima`

### Γιατί χρειάζεται;

✅ **Για να μοιράζεσαι συγκεκριμένο άρθρο** στα social media (όχι όλη τη λίστα νέων)
✅ **Για να βρίσκει σωστά το Google** το συγκεκριμένο περιεχόμενο (καλύτερο SEO)
✅ **Για να μένει σταθερή η διεύθυνση** ακόμα κι αν αλλάξεις τον τίτλο μετά

### Κανόνες για slugs

- **Μόνο μικρά αγγλικά γράμματα**: `a-z`
- **Αριθμοί επιτρέπονται**: `0-9`
- **Παύλες αντί για κενά**: `-`
- **Όχι κενά, ελληνικά, σύμβολα, κεφαλαία**

### Παραδείγματα

| Τίτλος άρθρου | ✅ Καλό slug | ❌ Λάθος slug |
|---|---|---|
| 5o Πανελλήνιο 2026 | `5o-panellinio-2026` | `5o Πανελλήνιο 2026` |
| Παγκόσμιο Ρεκόρ Τελιούση | `teliousi-paghosmio-rekor` | `Teliousi_Record!` |
| Εγγραφές 2026-27 | `eggrafes-2026-27` | `εγγραφές 2026/27` |
| Νέος Παίκτης | `neos-paiktis` | `Νέος Παίκτης` |

### Tips

- **Άστο σε άσπρο** αν δεν θες ξεχωριστή σελίδα (το άρθρο θα εμφανίζεται στη λίστα νέων κανονικά αλλά δεν θα έχει δικιά του διεύθυνση)
- **Άλλαξέ το** μόνο αν χρειαστεί. Αν αλλάξεις slug σε ένα άρθρο που έχεις ήδη μοιραστεί, το **παλιό link θα σπάσει**.
- **Κράτα το σύντομο** αλλά περιγραφικό. Όχι πάνω από 5-6 λέξεις με παύλες.
- Για **αθλητές**: συνηθίστε `firstname-lastname` ή σκέτο επίθετο (`panagiotidis`)

## 11. Troubleshooting

### «Δεν με αφήνει να μπω»
- Επιβεβαίωσε ότι συνδέεσαι με τον σωστό GitHub λογαριασμό (`KOKalamarias`)
- Αν είσαι σε άλλο GitHub λογαριασμό, κάνε logout από github.com και ξαναπροσπάθησε

### «Έγραψα κάτι αλλά δεν φαίνεται στο site»
- Επιβεβαίωσε ότι πάτησες **Publish now** (όχι μόνο Save)
- Περίμενε **2 λεπτά** πριν τσεκάρεις
- Καθάρισε cache του browser: ⌘+Shift+R στον Mac (ή Ctrl+F5 σε Windows)

### «Έχει βγει μήνυμα error / κίτρινο banner»
- Συνήθως πατώντας Publish ξανά λύνεται
- Αν επιμένει: βγες (Logout πάνω-δεξιά → avatar → Log out) και ξαναμπες

### «Έσβησα κάτι κατά λάθος»
- Όλες οι αλλαγές υπάρχουν στο **GitHub history**. Επικοινώνησε με τον τεχνικό υπεύθυνο για επαναφορά.

### «Θέλω να βγάλω άρθρο χωρίς να το διαγράψω»
- Στην επεξεργασία του άρθρου, **ξετσέκαρε** το πεδίο **«Δημοσιευμένο»** και Publish. Θα κρυφτεί από το site αλλά θα μείνει στη βάση.

### «Δεν φαίνεται η νέα φωτό που ανέβασα»
- Φωτογραφίες πάνω από **5 MB** μπορεί να αργήσουν ή να αποτύχουν. Συμπίεσε στο https://tinypng.com.
